  🌱 SITE ÉVÉNEMENT — GUIDE D'INSTALLATION SUR IONOS
═══════════════════════════════════════════════════════

DEUX ADRESSES, DEUX PAGES
-------------------------
  tonsite.be/quiz/         → LES JOUEURS. Compte à rebours, quiz, classement.
                             Rien d'autre : aucun accès caché.
  tonsite.be/quiz/admin/   → TOI. Identifiant + mot de passe, puis la gestion
                             des questions, le classement et les codes bonus.

CONTENU DU DOSSIER
------------------
index.html        → la page des joueurs
admin/index.html  → la page d'administration
api.php           → le serveur (scores, codes bonus, questions)
data/.htaccess    → protège les fichiers de données contre l'accès direct
data/             → scores.json, codes.json et questions.json y sont créés
                    automatiquement (ne pas les mettre dans le dépôt Git)

INSTALLATION (5 MINUTES)
------------------------
1. Connecte-toi à ton espace IONOS → "Hébergement" → gestionnaire
   de fichiers (ou via FTP avec FileZilla).
2. Envoie TOUT le dossier tel quel dans le répertoire de ton site
   (souvent la racine, ou un sous-dossier genre /quiz si tu préfères
   une adresse comme tonsite.be/quiz).
3. Ouvre l'adresse dans ton navigateur → ça doit fonctionner direct.
   (PHP est activé par défaut sur les hébergements IONOS.)

LES QUESTIONS : PLUS BESOIN DE TOUCHER AU CODE
----------------------------------------------
Va sur tonsite.be/quiz/admin/ → onglet "Questions".
Tu ajoutes, modifies, réordonnes et supprimes tes questions,
tu coches la bonne réponse, tu cliques "Enregistrer".
Les joueurs voient les nouvelles questions immédiatement.

Elles sont stockées dans data/questions.json. Tant que tu n'as rien
enregistré, ce sont trois questions d'exemple qui s'affichent.

À PERSONNALISER DANS LE CODE (le reste)
---------------------------------------
Dans index.html :
  → les DATES si elles changent (bloc CONFIG en haut du script)

Dans api.php :
  → les CODES BONUS ($BONUS_CODES) — les mêmes que sur tes QR codes
  → les IDENTIFIANTS ADMIN ($ADMIN_ID / $ADMIN_PWD)

⚠️ Le mot de passe livré est "a" : parfait pour tes tests, beaucoup
trop faible pour le jour J. Il donne accès à la remise à zéro, donc
à l'effacement de tout le classement. Change-le avant le 29/07.

ADMINISTRATION
--------------
tonsite.be/quiz/admin/ — trois onglets :
  → Questions   : l'éditeur décrit ci-dessus
  → Classement  : tous les participants (score, bonnes réponses, temps),
                  retrait d'une personne, remise à zéro complète
  → Codes bonus : quels QR codes sont pris, par qui et quand

Les identifiants ne sont écrits dans AUCUNE page : api.php les
revérifie à chaque action. Le mot de passe n'est donc pas visible
via "afficher le code source", et personne ne peut appeler une
action d'administration sans le connaître.

REMETTRE LES SCORES À ZÉRO (pour tes tests)
-------------------------------------------
Bouton dans l'onglet "Classement" (double confirmation), ou :
  tonsite.be/quiz/api.php?action=reset&pin=LE_MOT_DE_PASSE
⚠️ À ne PAS faire après le 29/07, ça efface tout le classement !
Les questions, elles, ne sont pas touchées par la remise à zéro.

QR CODES BONUS
--------------
Génère un QR code par code (ex : avec un générateur gratuit en ligne)
qui affiche simplement le texte du code (FAMI-A7K2...).
Le premier qui l'entre dans le quiz le "grille" pour les autres.

TÉLÉ EN MODE KIOSQUE
--------------------
Sur le PC branché à la télé, lance Chrome ainsi (touche Windows + R) :
  chrome --kiosk https://tonsite.be
Plein écran verrouillé, sans barre d'adresse. Pense à désactiver
la mise en veille du PC et de la télé.

BON À SAVOIR
------------
• Le classement est commun à tout le monde et se rafraîchit
  automatiquement toutes les 30 secondes (pratique pour la télé).
• Une seule participation par prénom (vérifiée côté serveur).
• Les réponses du quiz sont dans le code de la page : un collègue
  très malin qui ouvre le code source pourrait les trouver.
  Pour un quiz interne bon enfant, c'est largement suffisant —
  et le mélange aléatoire des questions/réponses complique déjà
  la copie entre collègues.

LE JARDIN COLLECTIF 🌼 (nouveau)
--------------------------------
• Les points sont devenus des GRAINES. Le classement se base sur les
  graines RÉCOLTÉES au quiz : elles ne diminuent jamais.
• Après le quiz, chacun peut « planter ses graines » : il dépense son
  solde pour poser des plantes sur la grille commune (48 cases).
  Planter ne fait PAS reculer au classement.
• Catalogue et coûts : à modifier dans api.php ($PLANTES).
  🌼 20 · 🌷 35 · 💜 50 · 🌻 80 · 🌹 120 · 🌳 200 graines.
• Un joueur qui revient plus tard retrouve son solde avec son prénom
  (onglet « Le jardin » → « Retrouver mes graines »).
• Dans l'admin, onglet 🌼 Jardin : retirer une plante (le planteur est
  remboursé) ou vider tout le jardin (tout le monde récupère tout).
• La remise à zéro générale vide aussi le jardin.
• Stades de croissance (selon les graines récoltées) :
  🌰 0 · 🌱 50 · 🪴 100 · 🌿 150 · 🌸 200 · 🌳 260
